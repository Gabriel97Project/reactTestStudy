Os testes servem para testarmos nossa aplicação em diversos cenários, podemos testar se 
a apiicação esta funcionando como deveria, por exemplo, testar se um botão esta realmente
execultando uma tarefa que ele deveria execultar.

*EXISTEM 3 TIPOS DE TESTE*

1. TESTE UNITÁRIO:
- SÃO TESTES ONDE TESTAMOS APENAS UM COMPONENTE, POR EXEMPO UMA FUNÇÃO;
- NESSE TESTE NÃO CHAMAMOS APIS NEM AGENTES EXTERNOS;
- EM CASO DE TESTES QUE CONTENHAM APIS, PRECISAMOS FAZER UM MOCK DESSA API PARA TESTA-LA;

2. TESTE DE INTEGRAÇÃO:
- SÃO TESTES QUE TESTAM 2 OU MAIS COMPONENTS TRABALHANDO EM CONJUNTO;
- NESTE TESTE TAMBEM REALIZAMOS MOCKS DE APIS EXTERNOS;
- POR EXEMPLO: TENHO O ARQUIVO APP E DENTRO DELE IMPORTEI OUTRO COMPONENTE
QNDO EU FIZER O TESTE DO APP, TAMBEM VOU TESTAR SE O COMPONENT IMPORTADO TAMBEM
ESTA EXECULTANDO OQUE ELE DEVE FAZER;

3. TESTE END-TO-END(E2E):
- TESTAMOS FLUXOS COMPLETOS DO USUÁRIO;
- NESTES TESTES NOS COMUNICAMOS APIS E BANCOS DE DADOS;
- EXEMPLO: CYPRES;
Podemos escrever test com algumas ferramentas, por exemplo jest e react testing library
